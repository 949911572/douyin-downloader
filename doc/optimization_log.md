# 抖音下载器优化记录

## 文档说明

本文件记录抖音下载器的优化建议及其决策结果，便于后续参考和追溯。

---

## 已实现的优化

### 1. 脚本迁移与统一管理
- **建议**：将本地脚本迁移到项目仓库，使用相对路径
- **决策**：✅ 采纳
- **实施时间**：2026-06-04
- **效果**：统一管理，方便迁移，Trae 可直接编辑

### 2. 统一时间过滤机制（核心优化）
- **建议**：在分页获取作品列表时实时进行时间过滤，并合并时间过滤优化与增量更新模式
- **决策**：✅ 采纳
- **实施时间**：2026-06-04（初始），2026-06-05（重构合并）
- **效果**：显著减少 API 请求次数，代码逻辑更清晰，自动选择最优时间阈值

**统一逻辑说明：**
1. **时间阈值计算**：使用增量更新的 `last_scan_time` 作为过滤阈值
2. **实时过滤**：在分页过程中实时检查视频发布时间，一旦发现所有视频都早于阈值，立即停止分页
3. **高效过滤**：减少不必要的 API 请求，提高下载效率

### 3. 请求限流控制
- **建议**：在 API 请求之间添加随机延迟，避免触发平台限流
- **决策**：✅ 采纳
- **实施时间**：2026-06-04
- **效果**：每次请求延迟 0.8-2 秒，降低被封禁风险

### 4. 下载失败重试机制
- **建议**：支持配置重试次数和延迟间隔
- **决策**：✅ 采纳
- **实施时间**：2026-06-04
- **效果**：提高下载成功率，可配置参数

### 5. 配置文件校验
- **建议**：启动时校验配置文件完整性和合法性
- **决策**：✅ 采纳
- **实施时间**：2026-06-04
- **效果**：提前发现配置错误，减少运行时异常

### 6. 数据库判断优化
- **建议**：只判断数据库记录，不检查本地文件（本地文件会迁移）
- **决策**：✅ 采纳
- **实施时间**：2026-06-04
- **效果**：支持本地文件迁移，不影响下载判断

### 7. 日志自动保存
- **建议**：下载完成后自动保存日志文件
- **决策**：✅ 采纳
- **实施时间**：2026-06-04
- **效果**：便于查看完整下载记录，日志自动保存到 `logs/` 目录，按日期命名

### 8. 智能跳过机制（方案1，已移除）
- **建议**：检查用户最新视频时间，如果用户没有发布新视频则跳过详细扫描
- **决策**：❌ 移除
- **移除时间**：2026-06-05
- **原因**：逻辑不确定性太多，难以准确判断用户是否有新视频，容易误判

### 9. 批量用户状态检查（方案4，已移除）
- **建议**：先批量获取所有用户的最新状态（只获取基本信息），再决定是否需要详细扫描
- **决策**：❌ 移除
- **移除时间**：2026-06-05
- **原因**：依赖智能跳过机制，一并移除

### 10. 代码去重与操作合并
- **建议**：检查代码逻辑，找出重复操作并合并，提高代码效率和可维护性
- **决策**：✅ 采纳
- **实施时间**：2026-06-04
- **效果**：减少重复操作，提高代码效率和可维护性

**具体优化点：**
1. **短链解析和URL解析去重**：在 `main.py` 中只做一次解析，将结果传递给 `download_url`
2. **API客户端复用**：修改 `download_url` 接受已有的 `api_client` 参数，避免重复创建连接
3. **用户信息获取优化**：从 `get_user_post` 返回结果中提取用户信息，避免单独调用 `get_user_info`（减少一次API调用）

### 11. 跳过阈值配置可调
- **建议**：将跳过阈值从固定4小时改为可配置参数，支持手动修改
- **决策**：✅ 采纳
- **实施时间**：2026-06-05
- **效果**：用户可根据需求灵活配置跳过阈值，提高下载效率

**具体实现：**
1. **配置文件**：在 `config.yml` 中添加 `skip_threshold_hours` 配置项（默认4小时）
2. **参数传递**：`main.py` 从配置文件读取参数并传递给 `ScanRecordManager`
3. **逻辑调整**：`should_skip()` 方法支持动态阈值，设置为空或0则不限制跳过

### 12. 失败视频管理功能
- **建议**：增加一个目录，用来存放下载失败的视频列表，用户可以手动处理，如果视频不需要，可以在数据库里面标记该视频已下载可跳过
- **决策**：✅ 采纳
- **实施时间**：2026-06-05
- **效果**：方便管理下载失败的视频，支持手动处理和跳过标记

**具体功能：**
1. **自动记录失败视频**：下载失败时自动记录到 `data/failed_videos/` 目录，按日期创建文件
2. **列出失败视频**：`--list-failed` 命令列出所有未处理的失败视频
3. **标记已处理**：`--mark-processed <aweme_id>` 标记视频为已处理
4. **标记跳过**：`--mark-skipped <aweme_id>` 标记视频为跳过，并记录到数据库，后续下载将自动跳过

### 13. 下载结果输出格式优化
- **建议**：优化下载结果的输出格式，提供更清晰的统计信息
- **决策**：✅ 采纳
- **实施时间**：2026-06-04
- **效果**：用户可快速了解下载情况，便于排查问题

**具体优化：**
1. **解析链接明细**：显示解析成功和失败的链接列表
2. **下载统计**：显示总下载数、成功数、失败数、跳过数等统计信息
3. **失败链接汇总**：下载完成后汇总显示所有失败的链接，方便用户后续处理

### 13. 动态跳过阈值（方案A）
- **建议**：根据用户历史发布频率，动态调整跳过时间阈值
- **决策**：❌ 暂不采纳
- **原因**：用户是手动执行任务，不是自动化定时任务，之前已讨论过

### 14. 视频ID去重机制（方案B）
- **建议**：记录每个用户已下载视频的ID列表，在获取作品列表时只获取新视频
- **决策**：❌ 暂不采纳
- **原因**：目前的去重是根据时间，视频是根据时间发布的，增加该功能意义不大

### 15. 用户信息缓存（方案C）
- **建议**：缓存用户基本信息（昵称、头像等），减少获取用户信息的API调用
- **决策**：❌ 暂不采纳
- **原因**：目前的信息已经足够了，json也存不了图片

### 16. 智能分页策略（方案D）
- **建议**：根据用户视频数量动态调整分页参数
- **决策**：❌ 暂不采纳
- **原因**：效果小，不值得实施

### 17. 并发请求优化（方案E）
- **建议**：在批量检查阶段，使用并发请求同时检查多个用户的状态
- **决策**：❌ 暂不采纳
- **原因**：有风险，可能触发平台限流

## 暂不采纳的优化建议

### 1. 链接去重
- **建议**：启动时自动检测并去除重复链接
- **决策**：❌ 暂不采纳
- **原因**：链接数量目前不多，人工可管理

### 2. 失败链接记录与重试
- **建议**：将失败链接保存到文件，下次启动自动重试
- **决策**：❌ 暂不采纳
- **原因**：需要先人工确认内容，再决定是否下载

### 3. 命令行参数覆盖配置
- **建议**：支持通过命令行参数覆盖配置文件
- **决策**：❌ 暂不采纳
- **原因**：通常通过 Trae 执行，无需命令行参数

### 4. 下载速度限制
- **建议**：添加下载速度限制配置
- **决策**：❌ 暂不采纳
- **原因**：不限速，追求下载效率

### 5. 多账号 Cookie 管理
- **建议**：支持多个账号 Cookie，自动切换尝试
- **决策**：❌ 暂不采纳
- **原因**：Cookie 刷新不稳定，人工管理更可靠

### 6. 代理支持
- **建议**：添加 HTTP/SOCKS 代理配置
- **决策**：❌ 暂不采纳
- **原因**：当前网络环境无需代理

### 7. 下载队列持久化
- **建议**：将下载队列保存到文件，支持断点续传
- **决策**：❌ 暂不采纳
- **原因**：下载任务相对简单，无需断点续传

### 8. 通知提醒
- **建议**：下载完成后发送桌面通知或邮件提醒
- **决策**：❌ 暂不采纳
- **原因**：不需要通知提醒
