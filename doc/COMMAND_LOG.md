### 开发测试命令
- `python run.py -h` - 查看帮助信息
- `python run.py -v` - 运行程序（详细日志）
- `python run.py` - 运行程序（默认日志）

### 失败视频管理命令
- `python run.py --list-failed` - 列出所有失败视频
- `python run.py --mark-processed <aweme_id>` - 标记视频为已处理
- `python run.py --mark-skipped <aweme_id>` - 标记视频为跳过

### 配置相关命令
- `python run.py -c config.yml` - 指定配置文件
- `python run.py -p <path>` - 指定保存路径
- `python run.py -t <number>` - 指定线程数

## 常用场景

### 场景1：日常下载
```bash
python run.py
```
**意图**：使用默认配置下载视频

### 场景2：查看详细日志
```bash
python run.py -v
```
**意图**：启用详细日志输出，便于调试和问题排查

### 场景3：处理失败视频
```bash
# 1. 列出失败视频
python run.py --list-failed

# 2. 标记不需要的视频为跳过
python run.py --mark-skipped 7123456789012345678

# 3. 标记已处理的视频
python run.py --mark-processed 7123456789012345679
```
**意图**：管理下载失败的视频，避免重复尝试

### 场景4：测试新功能
```bash
python run.py -v
```
**意图**：启用详细日志，验证新功能是否正常工作

## 注意事项

1. **命令参数**：所有参数都可以组合使用，如 `python run.py -v -t 10`
2. **配置文件**：默认使用 `config.yml`，可以通过 `-c` 参数指定其他配置文件
3. **日志级别**：`-v` 参数启用详细日志，`--show-warnings` 显示警告信息
4. **失败管理**：失败视频管理命令不需要运行下载，可以独立执行
