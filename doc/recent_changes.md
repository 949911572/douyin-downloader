## 2026-06-06

### 1. 修复 last_video_time 传递问题
- **问题**：`last_video_time` 在 `mode_result` 中有值，但没有传递到最终的 `result` 对象
- **修复**：在 `user_downloader.py` 的 `download()` 方法中添加 `result.last_video_time = mode_result.last_video_time`

### 2. 新增刷新 last_video_time 功能
- **功能**：提供命令跳过4小时限制，直接刷新所有用户的 `last_video_time`，不进行任何下载
- **具体实现**：
  - 在 `UserDownloader` 类中添加 `refresh_last_video_time()` 方法：只调用抖音 API 获取第一页数据（count=1），提取最新视频时间
  - 在 `ScanRecordManager` 类中添加 `update_last_video_time()` 方法：只更新时间字段，保留原有下载统计数据
  - 在 CLI 中添加 `--refresh-video-time` 命令：
    - `--refresh-video-time` - 刷新所有用户的 last_video_time
    - `--refresh-video-time "https://v.douyin.com/xxx/"` - 刷新指定用户的 last_video_time
  - 自动跳过4小时限制（通过设置 `skip_threshold_hours=0`）
  - 不使用 `start_time` 配置，始终获取最新视频时间

## 2026-06-05

### 1. 失败视频管理功能
- **功能**：增加目录存放下载失败的视频列表，支持手动处理和跳过标记
- **具体实现**：
  - 自动记录失败视频到 `data/failed_videos/` 目录，按日期创建文件
  - `--list-failed` 命令列出所有未处理的失败视频
  - `--mark-processed <aweme_id>` 标记视频为已处理
  - `--mark-skipped <aweme_id>` 标记视频为跳过，并记录到数据库

### 2. 下载结果输出格式优化
- **功能**：优化下载结果的输出格式，提供更清晰的统计信息
- **具体实现**：
  - 显示解析成功和失败的链接列表
  - 显示总下载数、成功数、失败数、跳过数等统计信息
  - 下载完成后汇总显示所有失败的链接

### 3. 重构合并时间过滤机制
- **功能**：合并时间过滤优化与增量更新模式，统一逻辑
- **具体实现**：
  - 统一时间过滤阈值计算（用户配置的 `start_time` 或增量更新的 `last_scan_time`）
  - 自动选择更大的时间戳（更晚的时间）作为过滤阈值
  - 在分页过程中实时检查视频发布时间，发现所有视频都早于阈值时立即停止分页
  - 减少代码重复，提高可维护性

### 4. 移除智能跳过机制和批量用户状态检查
- **移除原因**：逻辑不确定性太多，难以准确判断用户是否有新视频，容易误判
- **具体移除**：
  - 移除 `get_user_latest_video_time()` 方法
  - 移除批量检查用户状态的逻辑
  - 简化 `should_skip()` 方法，只基于"N小时内已成功处理"的简单判断

### 5. 跳过阈值配置可调
- **功能**：将跳过阈值从固定4小时改为可配置参数
- **具体实现**：
  - 在 `config.yml` 中添加 `skip_threshold_hours` 配置项
  - 支持手动修改跳过阈值（默认4小时）
  - 设置为空或0则不限制跳过，每次都会重新扫描

### 6. 修复 last_video_time 记录逻辑
- **问题**：`last_video_time` 字段总是被设置为扫描时间，而非实际视频发布时间
- **修复**：
  - 修改 `scan_record_manager.py` 的 `update_record()` 方法
  - 只有在有新视频时才更新 `last_video_time`，否则保留原有记录
  - 修复前：`last_video_time: last_video_time or now`（空值时用当前时间）
  - 修复后：`last_video_time: last_video_time if last_video_time else existing_record.get('last_video_time', now)`（保留历史记录）

### 7. 修复 sec_uid 为空问题
- **问题**：`scan_records.json` 中部分记录的 `sec_uid` 字段为空
- **修复**：在 `user_downloader.py` 的 `download()` 方法中添加 `result.sec_uid = sec_uid`
